# 工业表面质量识别任务概括

## 1. 任务背景

本任务面向工业材料表面图像的自动化视觉分析，目标是利用深度学习模型识别图像中需要关注的表面区域，并给出对应的类别信息。该任务可用于辅助质量检查、生产过程监测和结果统计分析。

为避免表述歧义，本文档统一使用“表面异常区域”“质量类别”“目标区域”等中性描述。

## 2. 任务目标

本任务的核心目标包括：

- 判断图像中是否存在需要关注的表面区域。
- 定位目标区域在图像中的位置。
- 识别目标区域所属的质量类别。
- 对不同模型的识别效果、定位效果和推理速度进行对比。

如果采用目标检测方法，模型输出为目标框、类别和置信度；如果采用分类方法，模型输出为整张图像的类别；如果采用分割方法，模型输出为目标区域的像素级轮廓。

## 3. 数据内容

数据集通常由表面图像和对应标注文件组成。

图像数据：

- 文件格式：`.jpg`、`.png`、`.bmp` 等常见图像格式。
- 图像内容：工业材料表面的局部视觉区域。
- 图像特点：背景相对单一，目标区域可能具有尺寸小、形状细长、边界不规则、对比度低等特点。

标注数据：

- 分类任务：每张图像对应一个类别标签。
- 检测任务：每张图像对应一个或多个目标框标签。
- 分割任务：每张图像对应一个像素级区域标注。

## 4. 推荐数据组织格式

目标检测任务建议使用 YOLO 格式组织数据：

```text
dataset/
  images/
    train/
    val/
    test/
  labels/
    train/
    val/
    test/
  data.yaml
```

其中，`images` 存放图像文件，`labels` 存放与图像同名的 `.txt` 标注文件。

## 5. YOLO 标注格式

每个 `.txt` 文件对应一张图像。文件中的每一行表示一个目标区域：

```text
class_id x_center y_center width height
```

字段含义如下：

- `class_id`：类别编号，从 `0` 开始。
- `x_center`：目标框中心点的横坐标，按图像宽度归一化到 `0-1`。
- `y_center`：目标框中心点的纵坐标，按图像高度归一化到 `0-1`。
- `width`：目标框宽度，按图像宽度归一化到 `0-1`。
- `height`：目标框高度，按图像高度归一化到 `0-1`。

示例：

```text
0 0.5123 0.4386 0.1842 0.0965
2 0.2310 0.7054 0.1188 0.0712
```

## 6. 数据配置文件

YOLO 训练通常需要一个 `data.yaml` 文件，用于描述数据路径和类别信息。

示例：

```yaml
path: dataset
train: images/train
val: images/val
test: images/test

names:
  0: category_0
  1: category_1
  2: category_2
  3: category_3
  4: category_4
  5: category_5
```

如果不希望在文档中出现具体类别名称，可以使用 `category_0`、`category_1` 这类中性命名方式。

## 7. 模型选择

建议将 YOLO 系列模型作为主要方法，因为该类模型具有训练流程成熟、推理速度快、部署方便等优点，适合工业图像中的实时识别需求。

可选基线模型包括：

- `YOLOv8n` / `YOLOv8s`：稳定、轻量，适合作为基础对比模型。
- `YOLO11n` / `YOLO11s`：较新的 YOLO 系列模型，适合作为主模型或增强对比模型。
- `Faster R-CNN`：经典两阶段检测模型，适合作为精度型对照。
- `RT-DETR`：实时 Transformer 检测模型，适合作为现代检测模型对照。

如果只做课程设计或小规模实验，推荐选择：

```text
YOLOv8n
YOLOv8s
YOLO11s
Faster R-CNN
```

## 8. 训练目标

训练阶段的目标是让模型学习图像中的表面视觉特征，并完成类别判断和区域定位。常见训练设置包括：

- 输入尺寸：`640`、`1024` 或 `1280`。
- 训练轮数：根据数据规模设置，一般可从 `100` 轮开始。
- 批大小：根据显存大小调整。
- 数据增强：翻转、亮度变化、对比度变化、随机裁剪、尺度变换等。

对于小目标或细长目标，可以适当提高输入分辨率，或使用切图训练与切图推理。

## 9. 评估指标

建议从准确性和效率两个角度评估模型。

准确性指标：

- `Precision`：预测结果中正确目标的比例。
- `Recall`：真实目标中被正确识别的比例。
- `mAP@0.5`：IoU 阈值为 `0.5` 时的平均精度。
- `mAP@0.5:0.95`：多个 IoU 阈值下的综合平均精度。
- `per-class AP`：每个类别单独的平均精度。

效率指标：

- `FPS`：每秒处理图像数量。
- `Latency`：单张图像推理耗时。
- `Params`：模型参数量。
- `FLOPs`：模型计算量。

工业应用中通常需要重点关注 `Recall`，因为漏检会直接影响后续质量判断。

## 10. 预期输出

模型推理后应输出以下内容：

- 图像中目标区域的位置。
- 每个目标区域的类别编号或类别名称。
- 每个预测结果的置信度。
- 可视化结果图。
- 统计结果，例如各类别数量、平均置信度、推理耗时等。

检测任务的典型输出示例：

```json
[
  {
    "class_id": 0,
    "class_name": "category_0",
    "confidence": 0.92,
    "bbox": [120, 80, 260, 150]
  }
]
```

其中，`bbox` 表示目标框左上角和右下角坐标，格式为：

```text
[x_min, y_min, x_max, y_max]
```

## 11. 实验设计建议

建议按以下流程开展实验：

1. 整理图像和标注文件。
2. 划分训练集、验证集和测试集。
3. 检查标注格式是否正确。
4. 训练基础模型。
5. 训练多个基线模型。
6. 对比不同模型的准确性和推理速度。
7. 分析各类别识别效果。
8. 可视化典型成功样例和失败样例。

最终报告中可以从模型性能、速度、模型大小、类别差异和实际应用价值等角度进行分析。

## 12. 总结

该任务属于工业表面图像的视觉识别问题。若数据提供目标框标注，推荐使用 YOLO 系列模型作为主要方法，并结合 Faster R-CNN 或 RT-DETR 作为对比模型。若数据只提供图像级标签，则应采用分类模型；若数据提供像素级区域标注，则可以进一步尝试分割模型。

整体实验重点应放在数据格式规范、模型对比充分、评估指标完整以及结果可视化清晰这几个方面。
